Next-Gen Rig Standard
OpenUSD · Face Tracking · Maya · Python · GitHub Actions

The goal was to deliver truly interoperable avatars by redesigning the production rig from the ground up: eliminating custom tool dependencies, resolving cross-generation inconsistencies, and establishing a single canonical standard the entire pipeline could rely on.

Architecture

The rig was designed as a real-time and USD-first specification covering model, skeleton, skin weights, blendshapes, and proxy. Facial expressions were migrated from a limited legacy blendshape set to a full Face Tracking expression schema, enabling more natural and consistent results across all avatar types. USD template file structures were defined for humanoid and mammal character types and adopted as the autorigging pipeline standard.

With the new standard established, the next challenge was bringing legacy work up to date. A batch migration across 1,500+ wearable assets, spanning multiple past avatar generations, rescued years of existing work and made it production-ready again. The migration made assets generation-agnostic, making them reusable across future avatar types without being tied to outdated pipeline assumptions.

Maya Autorigging Tool
+ USD Export API
Python · PySide2/6 · OpenUSD · MayaUSD · GitHub Actions API
Architecture

I built the tool as a PySide2 dockable Maya panel. A sequential workflow driving a modular USD Export API underneath. Each stage (mesh load, skeleton, skinweights, blendshapes, proxy, export, publish) is an independently-skippable module the UI calls into, so users can bypass stages while artists follow the linear flow.

This production Maya tool drives the complete avatar rigging pipeline from a single interface, eliminating the need to context-switch between scripts, file browsers, or external tools. The workflow is structured across phases: scene setup and topology validation against the template, skeleton import and RBF-based joint retargeting that fits joints to any custom template-based mesh, skin weight and blendshape loading, as well as blendshape and skinweight transfer with per-mesh overwrite control, cleanup optional stage where the user can load a control rig to QA deformartions and lastly, an export preparation that strips the control rig down to pure geometry and deformers for a modular USD export pipeline, and finally a CI publish step that dispatches directly to GitHub Actions via REST API.

Pipeline Flow

The USD export phase is its own 10-step modular API. Each step is independently skippable based on data presence. It builds a fully layered avatar.usd with discrete sublayers for geometry and materials, skeleton hierarchy, skin weights, blendshapes, and proxy mesh, handles shader conversion from Unity MetallicSmoothness to usdPreviewSurface via OpenCV, and packages the whole asset into a self-contained avatar.usdz with all textures bundled for AR and real-time rendering. On top of that, a sparse iteration layer system allows artists to diff their current Maya session against a locked published base and write only the changed prims to iteration.usd; so the canonical asset is never touched.

03 — ML Data Systems
Facial Landmarks
+ Synthetic Geometry
Houdini TOPS · OpenUSD · Python (hou API) · OpenCV · NumPy · COCO keypoints
The Challenge

The ML team needed 3D facial landmark ground truth data for both humanoid and mammal character types across diverse ethnic groups — data that didn't exist and couldn't be purchased off the shelf. The landmark specification had to be rigorous enough for CNN training while being naturally embedded in the same USD asset pipeline the art team used daily.

Architecture

I designed and encoded the full landmark specification as UsdGeom.Points primitives stored directly within each avatar USD asset — jaw, brow, eye contour, lip, and pupil sets for human and mammal. This meant ML engineers could consume training data directly from the USD pipeline without any conversion step.

  • Extended COCO keypoint schema from 17 to 57 joints with bilateral hand skeleton — delivered as ML training ground truth
  • Houdini TOPS batch network generating 6,000+ unique facial geometry variants from 500 base head targets
  • Procedural blending across 5 ethnic groups using zone-based blendshape mixing — fully automated, no manual sculpting per variant
  • OpenCV + NumPy post-processing for consistent UV and normal alignment across all generated variants

04 — Tooling
Production Maya Toolset
Python · maya.cmds · MayaUSD · PySide2/6 · OpenCV · NumPy
The Challenge

Three recurring friction points in the pipeline were slowing the team down: (1) batch refitting wearable assets to updated base meshes was manual and error-prone, (2) exporting shader networks to USD required manual mapping every single time, and (3) blendshape target weights were inconsistently normalized across the library, causing visual pops at runtime. Each needed a dedicated, artist-facing solution.

Tools Built
  • UAPT — Universal Avatar Pipeline Tool: Batch wearable refit pipeline running dry-run validation, JSON asset metadata autofill, and QC turntable generation. Processes a full library of assets overnight without supervision.
  • Material → USD Converter: Walks any Maya shader network graph and maps each node to its usdPreviewSurface equivalent. On-the-fly Unity MetallicSmoothness → glTF ORM texture conversion via OpenCV/NumPy baked in.
  • Blendshape Calibrator: Normalizes blendshape target weights across an entire library via dual list/tree view. Targets are validated, clamped, and rewritten in-place. Maya 2024 and 2025 compatible.

MediaPipe → Maya Facial Retargeting
Python · MediaPipe · Maya · ARKit blendshapes · JSON

Built a personal R&D tool that converts video footage into facial animation data for 3D rigs.

Facial performance capture usually requires expensive mocap hardware or dedicated ARKit-enabled iOS devices. I built a lightweight Python pipeline that removes that barrier entirely, extracting production-ready facial animation data from any standard MP4 file.

The tool uses Google's MediaPipe FaceMesh to track 468 facial landmarks per frame, mapping the resulting deltas to 52 ARKit blendshape weights. The pipeline then remaps those values to a custom rig's naming convention (blendshape targets or control names) and exports the data as JSON. A Maya script reads that output and automatically sets keyframes across all blendShape nodes, transferring a real human performance directly onto a 3D character rig's animation curves in seconds. The result is a practical, hardware-free alternative to traditional facial mocap, built for flexibility and speed.
